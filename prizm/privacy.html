#] PRIVACY ;]


Very simple privacy policy.

#] Definitions

  • "data removal" is the guild ID used for deleting the data corresponding to your server when you kick PRIZM from your server. Guild name is never stored
  • "guild" is referred to as "server" in the app but "guild" in the API. Frankly, I like the way "guild" sounds better so I'll use that.
  • "[...] name/content NOT stored" means that the name or content is quite literally not stored. I don't have a need to store it, and I want to keep your data anonymous

#] What's Stored

  • Reaction Roles
    • Guild ID "data removal"
    • Emoji string <:emoji_name:123456789012345678>
    • Role ID role name is NOT stored
    • Message ID message content is NOT stored
  • Guild info Will be removed in the rewrite
    • Guild ID
    • Guild name
  • Starboard [Starred Messages]
    • Starboard ID message content is NOT stored
    • Message ID message content is NOT stored
    • Guild ID is NOT stored since this stuff is never deleted. It would be annoying for all the messages to be restarred
  • Starboard [Settings]
    • Channel ID channel name is NOT stored
    • Emoji string any emojis used to star a message
    • Star count just a number
    • Whether or not users can use multiple stars ['multi stars']
    • Guild ID "data removal"
  • Starboard [Users]
    • User ID user names are NOT stored
    • Guild ID "data removal"
    • Received stars just a number, total stars
    • Given stars just a number, total stars
    • Messages starred how many messages made it on the starboard, just a number
  • Twitter
    • Tweeters tweeter names are NOT stored, only tweeter IDs as provided by the twitter API
    • Channel IDs channel names are NOT stored
    • Guild ID "data removal"
    • Watch/publish settings for tweets, replies, retweets or quote tweets
  • Chat bot
    • Message content, anonymously
    • User temporarily, used for context. Your username is associated with PRIZM ;]'s last message, never your own
  • FUTURE: Polls will be converted to interactions in the future
    • Message ID message content is NOT stored
    • User ID to prevent spamming the up/downvote buttons. Your user ID just assigns your vote. User name is NOT stored.
  • Logging settings
    • Whether or not a certain logging type is enabled
    • Channel ID channel name is NOT stored
    • Guild ID "data removal"

#] How to delete data

All your data is deleted once PRIZM ;] leaves your server. When PRIZM ;] rejoins, you have to set everything back up

#] How can I see my data

Currently, I do not have a command allowing you to download your data. However, you can be sure that I'm not collecting too much

#] Where does my data go

It stays on PRIZM ;]'s hard drive. No data is sent anywhere, even for personalized advertising or whatever those nasty bots do.
Your data is used to ensure proper function of the bot, it's really only preferences and stuff.
If you want more information, feel free to email me with any questions.

IMPORTANT NOTICE: PRIZM ;] is currently being rewritten to use /commands, so this privacy policy is subject to change to be more transparent about what data is being collected. Frankly, PRIZM ;] has so many commands that I forget what commands do what, but I assure you all data is anonymous other than the guild ID attached to guild name. After the rewrite, the guild name won't be stored either.
